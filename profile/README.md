## 为什么搬瓦工IP会被封？

先搞清楚原因，才能知道怎么防。

搬瓦工的IP被封，本质上是该IP地址被国内防火长城列入了屏蔽名单。这种封锁发生在网络层，搬瓦工的服务器自身一切正常运转，VPS面板也显示在线，但来自国内的连接请求会在网络层被直接丢弃。

常见的触发原因：

**流量特征异常**：这是大多数人被封的真正原因。VPS上运行的应用如果产生了某些敏感流量特征，就可能让IP进入黑名单。

**IP本身的历史问题**：有时候你刚买的VPS，前任用户就已经把IP搞臭了，相当于接手了一个"带病IP"。这种情况相对少见，但确实存在。

**运营商和时段的影响**：电信、联通、移动对IP封锁的敏感程度不一样。有些IP只是在某个运营商的网络下不通，换一个运营商的网络反而正常。

**重要提示**：IP被封不等于VPS被封，也不等于账号被封。服务器还在跑，数据还在，只是国内的网络路由到不了而已。处理方式有几种，后面详细说。

## 搬瓦工查看IP是否被封：三种方法详解

### 方法一：搬瓦工官方KiwiVM检测工具（最权威）

这是最直接、最权威的方法，搬瓦工官方提供的。

**操作步骤：**

**第一步**：登录搬瓦工账号，进入 Services → My Services，找到你的VPS，点击进入KiwiVM控制面板。

**第二步**：在KiwiVM面板里，找到你的VM ID（首页就能看到，是一串数字）。

**第三步**：在浏览器地址栏输入以下地址（把`你的VM_ID`替换成实际数字）：


https://kiwivm.64clouds.com/你的VM_ID/main-exec.php?mode=blacklistcheck


**第四步**：页面加载后，点击"Test Main IP"按钮，等待几秒钟出结果。

**结果解读：**

- **IP NOT BLOCKED**：IP正常，没有被封。如果你还是连不上，问题在别处（VPS没开机、系统崩了、流量耗尽等）。
- **IP BLOCKED**：IP已被封锁。这时候就需要考虑更换IP或者其他解决方案了。

有一点要注意：官方说这个检测在中国高峰时段可能不够准确，建议在不同时段测试2-3次再下结论，别因为一次"BLOCKED"就立刻花钱换IP。

### 方法二：port.ping.pe 第三方多节点检测（最直观）

这个工具是搬瓦工官方推荐的第三方检测方案，能同时从全球多个节点测试你的IP，国内外节点分开显示，结果一目了然。

**操作步骤：**

1. 打开 `port.ping.pe`
2. 在输入框输入：你的IP:端口（例如 `1.2.3.4:22`，22是SSH默认端口，根据你的实际端口填）
3. 点击Go，等待结果

**结果解读：**

- 上半部分是全球节点，下半部分是中国大陆节点
- 绿色 = 可访问，红色 = 不可访问
- **上半绿、下半红**：IP被封了，国内访问被墙，但国外正常
- **全绿**：IP完全正常
- **全红**：VPS本身有问题，不是IP被封

这个方法的好处是可视化强，哪些省份通、哪些省份不通，一眼就看出来。

### 方法三：国内端口扫描 + 国外端口扫描对比法（传统方法）

这是比较传统的判断方式，适合懒得登录KiwiVM的时候快速判断。

**基本逻辑：**
- 先在国内扫描你的IP和端口（SSH端口）
- 再用国外的工具扫同一个IP和端口
- 对比结果

**具体操作：**

**国内扫描**：打开 `tool.chinaz.com/port/`，输入VPS的IP和SSH端口，记录结果（R1）。

**国外扫描**：打开 `yougetsignal.com/tools/open-ports/`，输入同样的IP和端口，记录结果（R2）。

**判断逻辑：**

| R1（国内）| R2（国外）| 结论 |
|---------|---------|------|
| 开放 | 无所谓 | IP完全正常 |
| 关闭 | 开放(open) | **IP被封了** |
| 关闭 | 关闭(closed) | VPS自身问题，检查开机状态 |

## 不同情况的详细分析

### 情况1：IP能ping通，但SSH连不上

这是2018年以后越来越常见的情况。以前判断IP是否被封，就是直接ping，ping通就没封，ping不通就封了。但现在防火长城的封锁方式更精细了——它可以只封特定端口（比如TCP的22端口），让ICMP（ping用的协议）继续通。

所以你发现能ping通，但SSH死活连不上，这时候不能说IP没被封。可能是端口被封了，也可能是特定协议的流量被拦截。

这种情况用port.ping.pe检测最有用，它可以分别测ICMP和TCP，一下就能看出来。

### 情况2：IP完全不通，国内国外都ping不了

这种情况通常不是IP被封，而是VPS本身的问题：没有正常开机、系统崩溃、内存不足导致内核挂了，或者硬件故障。

处理方式：登录KiwiVM，检查VPS的Running状态，必要时重启或重装系统。

### 情况3：某个运营商不通，其他运营商正常

这说明IP只在某个运营商的网络下被封了，另一个运营商还没有同步这条封锁规则。这时候可以换个网络环境测试，如果真的只是单一运营商不通，等待一段时间可能自动解封。
